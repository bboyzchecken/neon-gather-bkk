# Cozy Avenue — Bar Social Layer Design
### ของสะสม / กิจกรรมมือว่าง / ระบบแต้มหัวใจ

---

## 1. หลักคิดพื้นฐาน

### ของสะสมสายบาร์ ≠ ของสะสมสายตกปลา

| | ตกปลา | บาร์ |
|---|---|---|
| พิสูจน์อะไร | ฝีมือ / ดวง | **คุณเคยอยู่ตรงนั้น กับใคร ตอนไหน** |
| เล่นคนเดียวได้ไหม | ได้ | **ไม่ควรได้** |
| แก่นอารมณ์ | ความสำเร็จ | ความทรงจำ |

> **กฎเหล็ก:** ของสะสมสายบาร์ต้อง **ได้มาเมื่อมีคนอื่นอยู่ด้วยเท่านั้น**
> ถ้าฟาร์มคนเดียวได้เมื่อไหร่ ความเป็น community ตายทันที

### กฎรอง
- ของสะสมจะมีความหมายก็ต่อเมื่อ **มีที่โชว์และมีคนเห็น** — ไม่มีตู้โชว์ = คนเลิกสะสมใน 1 สัปดาห์
- กิจกรรมในบาร์ต้อง **ไม่แย่งความสนใจจากบทสนทนา** (หยุดกลางคันได้เสมอ)

## 2. ของสะสมหลัก (เรียงตามความคุ้มค่า)

### 2.1 ที่รองแก้ว (Coaster) — ตัวเด่นที่สุด ★
บาร์จริงมีวัฒนธรรมสะสม coaster อยู่แล้ว และเข้ากับระบบที่มีอยู่แบบพอดีเป๊ะ

- **Dev cost แทบเป็นศูนย์** — ภาพ 2D ขนาดคงที่ (256×256) เพิ่มเป็น template slot อีกช่องในระบบอัปโหลดที่มีอยู่แล้ว
- **เจ้าของร้านออกแบบเอง** → UGC ที่ทำให้แต่ละร้านมีเอกลักษณ์
- **ขับ traffic ข้ามชั้น** → อยากได้ครบต้องเดินทุกร้านทุกชั้น (แก้ปัญหา "ชั้นบนร้าง")
- **เทรดได้** ผ่าน marketplace ที่มีอยู่แล้ว

**Rarity tier**
| Tier | เงื่อนไขได้มา |
|---|---|
| Standard | สั่งเมนูอะไรก็ได้ที่ร้านนั้นครั้งแรก |
| Seasonal | เฉพาะช่วงอีเวนต์/ฤดูกาล |
| Regular | ครบสถานะขาประจำของร้านนั้น |
| **Opening Night** | **7 วันแรกที่ร้านเปิดเท่านั้น — หลังจากนั้นหายตลอดกาล** |

#### ★ Killer feature: Opening Night Coaster
แก้ปัญหาที่ยากที่สุดของ player-driven marketplace: **ร้านเปิดใหม่ไม่มีลูกค้า**
ผู้เล่นจะแห่ไปหาร้านใหม่เอง เพื่อล่าของที่จะไม่มีอีกแล้ว → ร้านใหม่ได้ traffic ฟรีอัตโนมัติ โดยระบบไม่ต้องอุ้ม

### 2.2 สมุดเมนู / Tasting Passport
ลองเครื่องดื่มแต่ละตัว = ปั๊มตราลงสมุด โชว์ % ความคืบหน้า
เมนูเป็นของที่ผู้เล่นสร้างเอง → **ยิ่งมีร้านเยอะ สมุดยิ่งลึก โดยทีมไม่ต้องทำ content เพิ่มเลย**

### 2.3 สถานะ "ขาประจำ" (Regular)
สั่งเมนูเดิมที่ร้านเดิมครบ 20 ครั้ง → บาร์เทนเดอร์จำออเดอร์คุณได้ + ได้ title "ขาประจำร้าน…"
- ผู้เล่นได้ความรู้สึก **เป็นส่วนหนึ่งของที่นั่น** (emotional core ของบาร์จริง)
- เจ้าของร้านเห็นรายชื่อขาประจำ = ได้เครื่องมือ retention ของตัวเอง
- Dev cost = counter ตัวเดียว

### 2.4 ชนแก้ว (Cheers Log)
กดชนแก้วกับคนตรงหน้า → เก็บ log "ชนแก้วครั้งแรกกับ [ชื่อ]" + สถิติรวม
ถูกมาก อบอุ่นมาก และ **ต้องมีคนอื่นถึงทำได้** (ตรงตามกฎเหล็ก)

### 2.5 แผ่นเสียง / Jukebox
สะสมเพลง (แต่งเอง/chiptune — ห้ามใช้เพลงลิขสิทธิ์เด็ดขาด) แล้วเปิดให้ทั้งร้านฟัง
→ คนที่นั่งอยู่ **ควบคุมบรรยากาศร่วมกัน** = shared experience ที่ไม่ต้องพิมพ์แชท

### 2.6 เรื่องเล่าจากบาร์เทนเดอร์
แวะร้านแต่ละครั้ง NPC เล่าเกร็ดหนึ่งชิ้น → สะสมเป็นสมุดเรื่องเล่า
บางเรื่องโผล่เฉพาะดึก / เฉพาะตอนโต๊ะเต็ม → ให้เหตุผลกลับมานั่งซ้ำในเวลาต่างกัน

---

## 3. กิจกรรมมือว่าง (Idle Activities)

**เกณฑ์:** หยุดกลางคันได้ / คนอื่นดูอยู่ก็สนุก / รอบละ 1-2 นาที

- **ปาเป้า, ทอยเต๋า, ไพ่ง่ายๆ** — แข่งเบาๆ ที่โต๊ะ
- **ตู้คีบตุ๊กตา / กาชาปอง** — ได้สติกเกอร์/coaster สุ่ม → ป้อนกลับเข้าระบบสะสม
- **ตู้อาร์เคดมุมร้าน** — มี leaderboard แยกต่อร้าน → เจ้าของร้านมีเหตุผลซื้อตู้มาตั้งเพื่อดึงคน

---

## 4. การเชื่อมกับระบบเดิม

### 4.1 บาร์ = ห้องรอที่มีความหมาย ★
มีระบบ **ตกปลาแบบ auto/idle** อยู่แล้ว → วางเบ็ด/กับดักไว้ที่ท่าเรือ แล้วเดินมานั่งดื่มรอ
ระหว่างนั้นคุย เก็บ coaster ชนแก้ว พอถึงเวลาก็ไปเก็บปลา

> การนั่งเฉยๆ กลายเป็น **ช่วงเวลาที่มี progress เดินอยู่เบื้องหลัง** โดยไม่ต้องกดอะไรเลย
> — นี่คือคำตอบที่ตรงโจทย์ "นั่งเฉยๆ ไม่มีอะไรทำ" ที่สุด

### 4.2 เบื่อ → รับกะ
ระบบพนักงานผู้เล่นที่ออกแบบไว้เป็นคำตอบเดียวกัน — คนที่นั่งเฉยๆ กดรับกะเสิร์ฟได้เลย มีรายได้ + มีปฏิสัมพันธ์

### 4.3 ตู้โชว์ของสะสม
ต่อยอดจาก "ตู้ปลาโชว์" — ชั้นวางในร้าน/โปรไฟล์ โชว์ coaster ทุกใบ รูปถ่ายกับเพื่อน แผ่นเสียงหายาก

---

## 5. ★ ระบบภารกิจพิเศษ — แต้มหัวใจ (Heart System)

### 5.1 คอนเซปต์
พนักงาน NPC หญิง/ชาย ที่วาดโดยนักวาดมืออาชีพ **หมุนเวียนเข้ากะ** ในร้านต่างๆ ของ avenue
ผู้เล่นสานสัมพันธ์ผ่านการสั่งเมนู ให้ทิป ให้ของขวัญ และทำภารกิจส่วนตัวของแต่ละคน
→ สะสม **แต้มหัวใจ (Heart Points)** ปลดล็อกเนื้อเรื่อง ของรางวัล และภาพวาดพิเศษ

### 5.2 ⚠️ กติกาสำคัญที่สุด: NPC เท่านั้น
> **ห้ามใช้ระบบแต้มหัวใจกับผู้เล่นจริงเด็ดขาด**
>
> ในดีไซน์เดิมมี "พนักงานที่เป็นผู้เล่นจริง" อยู่ด้วย — ถ้าเอาระบบสะสมแต้มหัวใจไปครอบคนจริง
> มันจะกลายเป็นช่องทาง harassment ทันที (คนไล่ตามสะสมแต้มกับผู้เล่นคนอื่นที่ไม่ยินยอม)
>
> **แยกให้ขาด:**
> - พนักงาน NPC → มีระบบหัวใจ / เนื้อเรื่อง / ของขวัญ
> - พนักงานผู้เล่นจริง → มีแค่ระบบทิป, เพื่อน, รีวิว, เรตติ้ง เท่านั้น

### 5.3 พนักงานหมุนเวียน (Rotating Cast)
- ทุก season (2-3 เดือน) เชิญ **guest artist** วาดตัวละครใหม่ 1-2 คน
- ตัวละครเก่าไม่หายไป — ย้ายกะ / ย้ายร้าน / กลับมาในอีเวนต์
- แต่ละคนมี **profile**:

| ฟิลด์ | ตัวอย่าง | ผลต่อเกม |
|---|---|---|
| ร้านประจำ | บาร์ชั้น 2 | ดึงคนไปชั้นที่ร้าง |
| **เวลาเข้ากะ** | 20:00-24:00 เวลาจริง | **กระจายคนเข้าเกมทั้งวัน → server ไม่ร้างเป็นช่วงๆ** |
| เมนูถนัด | ค็อกเทลรสเปรี้ยว | ผูกกับระบบสั่ง-เสิร์ฟที่มีอยู่ |
| ของที่ชอบ / ไม่ชอบ | ชอบปลาหายาก, ไม่ชอบของทอด | **ขับ demand ให้ economy ทั้งระบบ** |
| บุคลิก / เรื่องเบื้องหลัง | — | เนื้อหาสำหรับ heart level |

> **ข้อสังเกต:** ฟิลด์ "เวลาเข้ากะ" ดูเล็ก แต่แก้ปัญหาใหญ่ที่สุดของเกม social —
> ถ้าทุกคนเข้าเกมเวลาเดียวกันหมด ช่วงอื่นจะร้าง การมีตัวละครที่เข้ากะดึกเท่านั้น = มีคนเข้าเกมตอนดึก

### 5.4 วิธีได้แต้มหัวใจ

| วิธี | ผูกกับระบบเดิม | หมายเหตุ |
|---|---|---|
| สั่งเมนูที่เธอ/เขาแนะนำ | ระบบสั่ง-เสิร์ฟ | แต้มน้อย ทำได้บ่อย |
| ให้ทิป | ระบบทิป | แต้มน้อย |
| **ให้ของขวัญ** | **ตกปลา / คราฟต์ / marketplace** | **แต้มเยอะสุด — ของยิ่งหายากยิ่งได้เยอะ** |
| คุยรายวัน | — | **จำกัดวันละครั้ง = retention hook** |
| ทำภารกิจส่วนตัว | ระบบเควส | ปลดล็อกตาม heart level |
| **นั่งในร้านตอนเข้ากะ** | — | **สะสมช้าๆ แบบ passive ตอนนั่งเฉยๆ** ← ตอบโจทย์ "ไม่มีอะไรทำ" โดยตรง |

> **จุดที่สวยที่สุดของระบบนี้:** ของขวัญต้องหามาจากระบบอื่นทั้งหมด
> อยากได้หัวใจเยอะ → ต้องไปตกปลาหายาก / ซื้อของจากร้านผู้เล่นคนอื่น / จ้างช่างคราฟต์
> → **ระบบจีบ NPC กลายเป็นเครื่องยนต์ที่ขับเศรษฐกิจของผู้เล่นทั้งเกม**

### 5.5 Heart Level & Reward Track

| Lv | ปลดล็อก |
|---|---|
| 1 | ทักทายทั่วไป |
| 2 | จำชื่อคุณได้ + บทสนทนาเฉพาะตัว |
| 3 | **Coaster ลายเฉพาะของตัวละคร** (เชื่อมระบบสะสม) |
| 4 | เล่าเรื่องส่วนตัว ตอนที่ 1 → เก็บลงสมุดเรื่องเล่า |
| 5 | **สูตรเครื่องดื่มลับ** → เอาไปใส่เมนูร้านตัวเองได้จริง |
| 6 | ของแต่งร้าน / cosmetic ที่เธอ-เขาให้ |
| 7 | ภารกิจส่วนตัว ตอนที่ 2 |
| 8 | มาเยี่ยมร้านของคุณเอง (event ที่คนอื่นเห็น) |
| 9 | เรื่องเล่าตอนจบ + title พิเศษ |
| **10** | **★ ภาพวาดพิเศษถ่ายคู่ในบูธถ่ายรูป + วางในตู้โชว์ได้** |

> **Lv10 = payoff ที่ออกแบบมาเพื่อ viral loop โดยเฉพาะ**
> ใช้บูธถ่ายรูปที่ออกแบบไว้แล้ว → ได้ภาพวาดคุณภาพสูงจากนักวาด → คนแชร์โซเชียลเอง

### 5.6 ⚠️ Guardrails (ต้องล็อกตั้งแต่ต้น)

1. **NPC เท่านั้น** — ห้ามครอบผู้เล่นจริง (ดูข้อ 5.2)
2. **เนื้อหาสะอาด (wholesome)** — เกม cozy บนเว็บ + มีผู้เล่นอายุน้อย ถ้าเนื้อหาล่อแหลมจะพัง brand ทั้งเกม และมีปัญหากับ payment provider/แพลตฟอร์มทีหลัง
3. **ตัวละครทุกคนเป็นผู้ใหญ่ชัดเจน** — สถานที่เป็นบาร์ ต้องระบุใน art brief ให้ชัดตั้งแต่วันแรก ไม่ปล่อยให้กำกวม
4. **ห้ามขายแต้มหัวใจตรงๆ** — จ่ายเงิน = หัวใจเพิ่ม จะเสี่ยงกฎหมาย gacha/loot box ในหลายประเทศ และทำลาย emotional core ของระบบ ขายได้แค่ cosmetic/ของขวัญที่ผู้เล่นหาเองได้อยู่แล้ว
5. **ไม่ลงโทษการจีบหลายคน** — นี่คือเกม cozy ไม่ใช่ drama sim ให้สะสมกับทุกคนพร้อมกันได้
6. **ตัวละครมีขอบเขต ไม่ใช่รางวัล** — เขียนบทให้มีความคิด/ความต้องการของตัวเอง ไม่ใช่ปลดล็อกได้ทุกอย่างถ้าจ่ายพอ (นี่คือความต่างระหว่างเกมที่คนรักกับเกมที่คนขยะแขยง)

### 5.7 Art Production Spec (ต่อ 1 ตัวละคร)

| Asset | จำนวน | หมายเหตุ |
|---|---|---|
| Portrait (ครึ่งตัว) | 6-8 สีหน้า | ปกติ / ยิ้ม / เขิน / ตกใจ / เศร้า / หัวเราะ |
| Full body | 1-2 ท่า | สำหรับหน้าโปรไฟล์ |
| Sprite ในโลกเกม | 1 ชุด | เดิน + เสิร์ฟ (chibi, ใช้ scale เดียวกับ avatar) |
| **ภาพพิเศษ Lv10** | 1 | ภาพคู่ในบูธถ่ายรูป — คุณภาพสูงสุด |
| Coaster art | 1 | ผูกกับระบบสะสม |
| ชุด/สกินเสริม | 2-3 | สำหรับ monetization |

> **นี่คืองานที่ AI gen ทำแทนไม่ได้** — ตรงกับที่คุยกันไว้ตั้งแต่ต้น
> โมเดล gen ภาพคุมความสม่ำเสมอของ "ใบหน้าเดิม" ข้าม 8 สีหน้าไม่ได้จริง
> แต่ข้อดีคือ scope ต่อตัวละครชัดเจนมาก → ตีราคาจ้างนักวาดได้แม่น และจ้างเป็นชิ้นได้

### 5.8 Monetization ที่ทำได้ (โดยไม่ขายหัวใจ)
- **ชุด/สกินให้ NPC** — เธอ/เขาใส่ชุดใหม่ตอนเข้ากะ คนทั้งร้านเห็น
- **ของขวัญพิเศษ** — แต่ต้องหาเองในเกมได้ด้วย ไม่ใช่ paywall
- **Art book / seasonal pack** — รวมภาพจาก guest artist
- **★ Guest artist collab = ช่องทางการตลาดในตัว** — นักวาดมีแฟนคลับของตัวเอง การเชิญนักวาดมาวาดตัวละครประจำ season ไม่ใช่แค่ต้นทุน art แต่คือ **user acquisition channel** ที่พาคนใหม่เข้าเกมทุกฤดูกาล

### 5.9 ทางเลือกขยายตลาด
โจทย์ตั้งต้นคือ "จีบสาว" แต่ระบบเดียวกันนี้รองรับตัวละครทุกเพศได้โดยไม่ต้องเขียนโค้ดเพิ่มเลย
ตลาด otome / BL ในเอเชียใหญ่มาก — ถ้าแต่ละ season มีตัวละครหลากหลาย จะได้ผู้เล่นเพิ่มเป็นเท่าตัวจากระบบเดิม (แค่เพิ่มค่า art)

```
CheersLog      (player_id_a, player_id_b, first_cheers_at, total_count)

StaffNPC       (id, name, artist_credit, home_shop_id, shift_start, shift_end,
                signature_menu_id, season, is_active)
StaffGiftPref  (staff_id, item_id, preference)         -- loved/liked/neutral/disliked
PlayerAffinity (player_id, staff_id, heart_points, heart_level,
                last_talked_at, unlocked_rewards[])
StaffStoryNode (staff_id, required_level, story_text, reward_type, reward_ref)
```

**จุดที่ต้องระวังตอน implement**
- `last_talked_at` ต้องเช็กด้วย server time เท่านั้น (กันคนหมุนนาฬิกาเครื่องตัวเอง)
- แต้มหัวใจต้องคำนวณฝั่ง server ล้วน ห้ามให้ client ส่งแต้มมาเอง
- `PlayerCoaster` ต้องมี unique constraint กัน duplicate จาก race condition ตอนกดรัวๆ

## 7. ลำดับความสำคัญ

| ลำดับ | ทำอะไร | เหตุผล |
|---|---|---|
| **1** | **Coaster + Opening Night Coaster** | ต้นทุนต่ำสุด แก้ 3 ปัญหาพร้อมกัน (คนนั่งเฉยมีเป้าหมาย / ร้านใหม่มีลูกค้า / ชั้นบนมีคนขึ้น) |
| 2 | ขาประจำ + ชนแก้ว | counter อย่างละตัว แต่ได้ emotional core |
| 3 | Tasting passport | content โตเองตามจำนวนร้าน |
| 4 | Idle fishing ↔ นั่งบาร์ | เชื่อมสองระบบที่มีอยู่แล้วให้กลมกล่อม |
| **5** | **Heart System (ตัวละครแรก 1 คน)** | ทำ 1 คนให้จบก่อน วัดผลจริงว่าคนติดไหม ค่อยขยายเป็น season |
| 6 | Jukebox, เรื่องเล่า, ตู้อาร์เคด | ambient content เพิ่มบรรยากาศ |

> **อย่าเปิด Heart System พร้อม 5 ตัวละครในรอบแรก** — ต้นทุน art สูงและยังไม่รู้ว่า loop เวิร์กไหม
> ทำคนเดียวให้สมบูรณ์ ดู retention/heart level distribution ก่อน แล้วค่อยตัดสินใจลงทุน season ถัดไป

## 8. ความเสี่ยงที่ต้องติดตาม

1. **Heart system กลืนเกม** — ถ้าคนสนใจแต่จีบ NPC จนไม่สนใจร้านของผู้เล่นด้วยกัน community จะตาย ต้องคุมสัดส่วน reward ให้ระบบผู้เล่น-ผู้เล่นยังคุ้มกว่า
2. **ต้นทุน art ต่อ season** — ตัวละคร 1 คน = งานวาดหลายชิ้น ต้องมีรายได้รองรับก่อนขยาย cast
3. **เส้นแบ่ง NPC / ผู้เล่นจริงเบลอ** — UI ต้องแยกให้ชัดว่าใครเป็น NPC ใครเป็นคนจริง ไม่งั้นผู้เล่นจะเผลอปฏิบัติกับคนจริงแบบเดียวกับ NPC
4. **Coaster เฟ้อ** — ถ้าร้านแจก coaster ง่ายเกินไป มันจะไร้ค่า ต้องมี cap ต่อร้าน/ต่อ season
